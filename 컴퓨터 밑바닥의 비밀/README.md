### 🗂 세부 커리큘럼
#### 1주차: 전체 오리엔테이션
- 범위: 1.1 ~ 1.5 (프로그래밍 언어, 컴파일러, 링커, 추상화)
- 공통: 코드 → 실행파일까지 큰 흐름 잡기
- FE: JS/Python 같은 인터프리터 언어와 비교, 프론트 런타임 이해
- BE: JIT, 바이트코드, JVM과 비교
- DevOps: 빌드/배포, 링커, 동적 라이브러리 → 컨테이너 이미지와 연계

#### 2~3주차: 운영체제와 실행 모델
- 범위: 2장 전체 (프로세스, 스레드, 코루틴, 동기/비동기, 이벤트 루프)
- 공통: 스레드/프로세스 차이, 동기/비동기/논블로킹 핵심
- FE: JS 이벤트 루프, 콜백 지옥, async/await → 책의 코루틴/비동기 부분 집중 (2.4~2.8)
- BE: 스레드 풀, 동시성 제어, 스레드 안전 패턴 (2.3, 2.7, 2.8)
- DevOps: 서버 아키텍처 (다중 스레드, 이벤트 루프), 고성능 서버 모델 (2.8)

#### 4~5주차: 메모리 구조와 버그
- 범위: 3장 전체 (메모리, 포인터, 스택/힙, 메모리 풀, 메모리 버그)
- 공통: 가상 메모리, 스택/힙 구조 큰 그림
- FE: JS/브라우저 엔진에서의 스택/힙 매핑 이해, 메모리 누수와 GC 연계 (3.1~3.3, 3.7)
- BE: malloc, 메모리 풀, GC 동작과 대비, 포인터 버그 사례 (3.4~3.6, 3.7)
- DevOps: 시스템 콜(brk/mmap), 메모리 풀 최적화, OOM, 메모리 리소스 관리 (3.5~3.6)

#### 6~7주차: CPU와 명령어 집합
- 범위: 4장 전체 (CPU 구조, 파이프라인, CISC/RISC, 스레드 전환, 인터럽트)
- 공통: CPU-메모리 상호작용, 인터럽트, 분기 예측
- FE: 성능 최적화와 CPU 파이프라인 직관 (4.4 if문/분기 예측)
- BE: 함수 호출/스택 프레임/시스템 콜(4.9), 멀티코어 최적화 감각
- DevOps: CPU 스케줄링, 하이퍼스레딩, CISC vs RISC → 클라우드 인스턴스 선택 관점 (4.2~4.8)

#### 8주차: 캐시와 동시성
- 범위: 5장 전체 (캐시, 메모리 일관성, 메모리 장벽)
- 공통: 캐시 지역성, 캐시 라인, 메모리 장벽
- FE: 캐시 친화적인 데이터 구조 설계 (5.2) → JS 엔진 최적화와 비교
- BE: 캐시 라인, 거짓 공유 문제, lock-free 프로그래밍 (5.3, 5.4)
- DevOps: CPU 캐시 일관성 문제, 멀티코어 성능 이슈, 분산 시스템 메모리 장벽 (5.1, 5.4)

#### 9주차: 입출력과 고성능 서버
- 범위: 6장 전체 (I/O, DMA, 파일 읽기, epoll, mmap)
- 공통: 동기 vs 비동기 I/O, epoll/select/poll 비교
- FE: 브라우저 이벤트 루프와 OS I/O 모델 비교 (6.1, 6.4)
- BE: epoll 기반 고성능 서버 구조, mmap vs read/write 차이 (6.4~6.5)
- DevOps: DMA, 인터럽트, 커널 버퍼링, epoll 시스템콜 튜닝 (6.2~6.6)

#### 10주차: 전체 리뷰 & 응용 토론
- 발표 주제 예시:
  - FE: “자바스크립트 이벤트 루프 vs OS 이벤트 루프”
  - BE: “스레드 안전한 코드 패턴과 캐시 친화적 데이터 구조”
  - DevOps: “epoll 기반 서버 구조와 Kubernetes Pod 리소스 관리 연결”

- BE → 언어 런타임, 스레드 안전, 메모리 관리, 캐시 최적화
- DevOps → CPU/OS/I/O 구조, 캐시 일관성, epoll, 시스템 콜
