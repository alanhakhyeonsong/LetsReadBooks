책에서 파생된 궁금한 사항이나 심화 주제를 `docs/`에 추가 학습 문서로 작성합니다.

## 입력
- $ARGUMENTS: "주제 (관련 책/챕터)" 형식 (없으면 사용자에게 질문)

## 절차

1. **주제 파악**: 사용자가 궁금해하는 핵심 주제 확인
2. **출처 확인**: 어떤 책/챕터에서 파생된 궁금증인지 확인
3. **조사 및 작성**: 주제에 대한 심화 내용을 조사하고 문서 작성
4. **파일 생성**: `docs/주제-kebab-case.md` 로 저장

## 문서 템플릿

```markdown
# 주제 제목

`책 이름` N장의 내용을 바탕으로, [주제]에 대해 추가 학습한다.

---

## 1. 배경 및 학습 동기
- 어떤 맥락에서 이 궁금증이 생겼는지

## 2. 핵심 개념
- 주제의 핵심 개념 정리

## 3. 실무 적용
- 코드 예시 (Kotlin/Java + Spring Boot 기반)
- 구조/흐름 다이어그램 (Mermaid)

## 4. 주의사항 및 트레이드오프
- 실무에서 고려할 점

## 5. 참고 자료
- 관련 링크, 공식 문서 등
```

## 파일명 규칙
- 위치: `docs/`
- 파일명: 영문 kebab-case (예: `async-exception-handling-strategy.md`)

## 커밋 컨벤션
```
docs: 문서 주제 요약
```
예시: `docs: 비동기 멀티홉 환경 예외 처리 전략`

## 참고
- 기존 문서(`docs/async-exception-handling-strategy.md`) 스타일을 참고
- Mermaid 다이어그램은 구조/흐름 설명이 필요한 경우 반드시 포함
- 코드 예시는 Kotlin/Java + Spring Boot 환경 기준
