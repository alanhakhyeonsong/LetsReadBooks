## 입력
- 책 이름: $ARGUMENTS (없으면 사용자에게 질문)
- 저자: (없으면 사용자에게 질문)
- 목차: 사용자가 직접 제공하면 그대로 사용. 제공하지 않으면 스켈레톤으로 생성.

## 절차

1. **디렉토리 생성**: `책 이름/contents/` 구조 즉시 생성
2. **README.md 작성**: 아래 템플릿으로 `책 이름/README.md` 생성
3. **루트 README.md 업데이트**: `## Books` 목록 끝에 새 책 추가 (End 표시 없음)
4. **결과 안내**: 생성된 구조와 다음 단계 안내

## README.md 템플릿

### 사용자가 목차를 제공한 경우
```markdown
# 책 이름

## 목차
- 1장. 사용자가 제공한 챕터 제목
- 2장. 사용자가 제공한 챕터 제목
- ...
```

### 사용자가 목차를 제공하지 않은 경우
```markdown
# 책 이름

## 목차
<!-- 목차는 챕터 정리 시 업데이트 -->
```

## 목차 작성 규칙
- 아직 정리하지 않은 챕터는 텍스트만, 정리 완료된 챕터는 `[N장. 제목](./contents/chapterXX.md)` 링크로 작성한다.
- 책에 Part 구분이 있으면 `### Part N - 파트 제목` 형식으로 구분한다.

## 중요: 웹 검색 금지
- 목차를 찾기 위해 웹 검색(WebSearch, WebFetch)을 하지 않는다.
- 사용자가 목차를 제공하지 않으면 스켈레톤으로 생성하고, 챕터 정리할 때마다 업데이트한다.
- 사용자가 명시적으로 "목차 검색해줘"라고 요청한 경우에만 웹 검색을 수행한다.

## 규칙
- 디렉토리명은 한국어 책 이름 그대로 사용 (기존 컨벤션)
- 커밋 메시지 제안: `chore: 책 이름 학습 시작`
