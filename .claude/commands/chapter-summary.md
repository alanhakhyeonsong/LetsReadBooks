책의 특정 챕터를 요약 정리하는 Markdown 문서를 작성합니다.

## 입력
- $ARGUMENTS: "책 이름 N장 챕터 제목" 형식 (없으면 사용자에게 질문)

## 절차

1. **대상 확인**: 책 이름, 챕터 번호, 챕터 제목 확인
2. **기존 파일 확인**: 해당 책 디렉토리(`책 이름/contents/`)의 기존 파일명 패턴 확인
3. **파일 생성**: 기존 패턴에 맞춰 `책 이름/contents/chapterXX.md` 생성
4. **내용 작성**: 사용자가 제공한 내용 기반으로 요약 정리

## 문서 템플릿

```markdown
# 챕터 제목

## 핵심 내용

### 소주제 1
- 요약 내용

### 소주제 2
- 요약 내용

## 정리
- 핵심 takeaway 요약
```

## 파일명 규칙
- 해당 책 디렉토리의 기존 파일명 패턴을 우선 따른다
- 기본값: `chapterXX.md` (예: `chapter01.md`, `chapter10.md`)

## 커밋 컨벤션
```
N장. 챕터 제목
```
예시: `3장. 코드에서 신경 써야 할 예외와 오류 처리 패턴`
