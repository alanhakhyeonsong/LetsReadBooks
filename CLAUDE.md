# LetsReadBooks - 프로젝트 규칙

## 프로젝트 개요
2021년부터 기술서적을 읽고 챕터별로 요약 정리하는 개인 학습 레포지토리.

## 디렉토리 구조
```
LetsReadBooks/
├── 책 이름/
│   ├── README.md              # 책 소개 및 목차
│   └── contents/
│       └── chapterXX.md       # 챕터별 요약 정리
├── docs/                      # 추가 학습 문서 (심화/실무 적용)
└── CLAUDE.md
```

## 핵심 규칙

### 1. 챕터 요약 정리
- 각 챕터는 `책 이름/contents/` 하위에 `.md` 파일로 작성한다.
- 파일명은 `chapterXX.md` 또는 챕터 제목을 사용한다.

### 2. 추가 학습 문서 (docs/)
사용자가 책을 읽다가 **궁금한 사항**이나 **추가 학습이 필요한 주제**가 있을 때, `docs/` 디렉토리에 심화 문서를 작성한다.

#### 문서 작성 규칙
- **파일 위치**: `docs/` 디렉토리 하위에 Markdown 파일로 생성
- **파일명**: 주제를 나타내는 영문 kebab-case (예: `async-exception-handling-strategy.md`)
- **출처 명시**: 어떤 책의 어떤 챕터에서 파생된 학습인지 문서 서두에 반드시 기록
- **내용 구성**:
  1. 주제 소개 및 학습 동기 (어떤 책/챕터에서 궁금증이 생겼는지)
  2. 핵심 개념 정리
  3. 실무 적용 관점의 분석 (코드 예시 포함 가능)
  4. 구조/흐름이 필요한 경우 Mermaid 다이어그램 포함
  5. 참고 자료 및 추가 학습 링크

#### 문서 작성 예시
```markdown
# 주제 제목

`책 이름` N장의 내용을 바탕으로, [주제]에 대해 추가 학습한다.

---

## 1. 배경 및 학습 동기
...

## 2. 핵심 개념
...

## 3. 실무 적용
...
```

### 3. 커밋 컨벤션
- 챕터 요약: `N장. 챕터 제목` (예: `3장. 코드에서 신경 써야 할 예외와 오류 처리 패턴`)
- 추가 학습 문서: `docs: 문서 주제 요약` (예: `docs: 비동기 멀티홉 환경 예외 처리 전략`)
- 기타 변경: `chore: 변경 내용`

### 4. 언어
- 모든 문서는 **한국어**로 작성한다.
- 파일명과 코드 예시 내 식별자는 영문을 사용한다.
